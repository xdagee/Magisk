## AGENTS.md for Magisk

### Toolchain setup

- `build.py` (repo root) is the single build orchestrator. Do **not** invoke `gradlew`, `cargo`, or `ndk-build` directly; use the matching `build.py` subcommand instead.
- When a raw toolchain command is unavoidable, prefix it with `python scripts/env.py <cmd>` so the custom NDK (`$ANDROID_HOME/ndk/magisk`), Rust sysroot, and JDK 21 are configured. `build.py` subcommands already do this internally.
- Required env: `ANDROID_HOME` (Android SDK path). JDK 21 (set `ANDROID_STUDIO` to its install path, or have `javac` on PATH).
- The NDK is a custom ONDK, not a standard Android NDK. Install/upgrade it with `python build.py ndk` (version pinned in `scripts/env.py` as `ondk_version`).
- Clone with submodules: `git submodule update --init --recursive`. Everything under `native/src/external/` is submodules — do not edit there.
- On macOS, install GNU make (`brew install make`, set `GNUMAKE=gmake`).

### Build commands (run from repo root)

- `python build.py native` — build Rust + C++ native binaries into `native/out/<abi>/`.
- `python build.py app` — build the main `:apk` module.
- `python build.py app-ng` — build the Compose `:apk-ng` module.
- `python build.py stub` / `python build.py test` — build stub / test APKs.
- `python build.py all` — native → app → app-ng → test (correct order).
- `python build.py clean [native|cpp|rust|app]` — clean outputs.
- Flags: `-r` release, `-v` verbose, `-c <file>` custom config (see `config.prop.sample`).
- **Native must be built before the app.** The Gradle `setupCoreLib` task syncs `native/out/<abi>/` into jniLibs and errors with "Please build binaries first!" if the binaries are missing. `build.py all` handles the order; running `app` alone requires a prior `native` build.

### Native (Rust + C++)

- Rust workspace at `native/src/Cargo.toml`; members: `base`, `base/derive`, `boot`, `core`, `init`, `sepolicy`. Edition 2024. `external/` is excluded from the workspace.
- C++ is built via `ndk-build` (Android.mk). Rust↔C++ FFI uses `cxx` (generates `*-rs.*pp` files).
- `python build.py cargo <cmd>` runs cargo against the workspace with the correct env. `python build.py clippy` runs clippy across debug + release for all ABIs.
- Clippy lints: `unwrap_used = "deny"` (workspace-wide). `rustfmt.toml`: `imports_granularity = "Module"`.
- `python build.py gen` regenerates `native/compile_commands.json` and Rust/C++ FFI bindings for IDE setup.

### App (Gradle, Kotlin/Java)

- The Gradle project root is `app/` (wrapper at `app/gradlew`, **not** repo root). Modules: `:apk` (XML Views + data binding), `:apk-ng` (Jetpack Compose + Navigation3), `:core` (shared library), `:shared`, `:stub`, `:stub-res`, `:test`.
- Prefer Kotlin for all new code.
- Custom Gradle DSL lives in `app/build-logic`: `setupMainApk` / `setupCoreLib` / `setupTestApk` in `Setup.kt`, build config in `Plugin.kt`.
- Version and signing config flows: `config.prop` (see `config.prop.sample`) + `app/gradle.properties` (`magisk.*` keys) → `app/build/flags.prop` (generated by `build.py`) → read by `Plugin.kt` `Config` object.
- compileSdk/targetSdk 37, minSdk 23, Java 21.
- After app changes, build the affected module(s) with `python build.py app` / `app-ng` / `stub` / `test` to verify.

### Testing

- Tests are instrumented (on-device), not host unit tests. UI/integration tests live in the `:test` module; `:core` declares test dependencies as `compileOnly`.
- `python build.py test` builds the test APK (forced to release to avoid classname clash).
- Emulator testing: `scripts/avd.sh` (API 23–37 matrix) and `scripts/cuttlefish.sh`; `python build.py emulator` provisions an AVD; `python build.py avd_patch <image> <output>` patches boot images.

### CI

- `.github/workflows/build.yml`: `build` (macOS, `./build.py -vr all` then `./build.py -v all`), `test-build` (Windows + Ubuntu, `python build.py -v -c .github/ci.prop all`), plus `avd-test`/`cf-test` matrices.
- The setup action (`.github/actions/setup`) installs JDK 21, Python 3, sccache, and runs `python build.py -v ndk` to install the NDK.
